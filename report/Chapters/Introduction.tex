\lhead{Introduction}
\chapter{Introduction}
\label{SecIntroduction}

The Traveling Salesman Problem (TSP) asks for the shortest closed tour that
visits every city in a given set exactly once before returning to the
starting city. The statement is simple, but the TSP is
NP-hard~\cite{applegate2006traveling} and its search space grows
factorially with the number of cities. Exhaustive enumeration is therefore
not practical for all but the smallest instances. The TSP has been widely
used as a standard testbed for combinatorial metaheuristics, and Genetic
Algorithms (GAs) are among the most commonly studied
approaches~\cite{holland1975adaptation,goldberg1989genetic}. In this
report we implement a GA for the static single-objective TSP, with
particular attention to how constraint violations are detected and
corrected during the search.

\section{Background}
\label{Intro:Background}

A tour in the TSP is a permutation of the city indices, and a permutation
is feasible only when every city appears exactly once. This makes operator
design especially important: ordinary crossover operators taken from
binary or real-valued GAs do not preserve the permutation property and
often produce offspring that contain duplicated cities while omitting
others. Specialised operators have been developed for this reason,
including Partially Mapped Crossover (PMX)~\cite{goldberg1985alleles},
Order Crossover (OX)~\cite{davis1985applying}, and Cycle Crossover
(CX)~\cite{oliver1987study}. When such operators are not used, the GA
must handle infeasibility in some other way, either by penalising it in
the fitness function or by repairing offspring before evaluation.

We use kroA100 from TSPLIB~\cite{reinelt1991tsplib} as our benchmark.
It contains 100 cities in the Euclidean plane with a known optimal tour
length of 21\,282. It is large enough to show meaningful differences
between constraint-handling strategies, yet small enough that
thirty-seed comparisons and parameter sweeps remain computationally
feasible.

\section{Problem Statement}
\label{Intro:ProblemStatement}

When a GA for the TSP uses a crossover operator that does not preserve
permutations (single-point crossover being the simplest case),
offspring will typically contain duplicated cities at some positions and
omit other cities entirely. Computing tour length on such a chromosome
gives a distance that does not correspond to any valid tour. Worse, the
value is often artificially low because the missing cities never
contribute to the sum. Selection then favours these infeasible
chromosomes, which pushes the population away from the feasible region.

There are three strategies to address this. First, feasibility can be
preserved by construction using permutation-aware operators such as PMX.
Second, infeasibility can be tolerated but discouraged through a penalty
term proportional to the violation count. Third, infeasibility can be
removed after recombination by a repair step that detects duplicates and
missing cities and rewrites the chromosome into a valid permutation. We
want to know how these three strategies compare on a fixed benchmark
when the rest of the GA pipeline is held constant.

\section{Contributions}
\label{Intro:Contributions}

We implement a full GA pipeline for the TSP using permutation encoding,
tournament selection ($k=3$), swap mutation, and elitism. The primary
configuration uses naive single-point crossover together with a
random-insertion repair operator that runs on every generation, so every
offspring is guaranteed feasible before evaluation. We then run a
three-arm comparison on kroA100 over 30 seeds, comparing repair against
a penalty function and against PMX. Repair and PMX produce similar mean
tour lengths, while penalty performs worse by a clear margin. A
1\,620-run parameter sweep adds to the comparison. We report convergence
curves, diversity traces, and tour plots together with a discussion of
where the GA works well and where it falls short.
