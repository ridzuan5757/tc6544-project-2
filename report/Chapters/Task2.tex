\section{Task 2: Algorithm Development}
\label{Task2}

\subsection{Genetic Algorithm Design (Task 2.1)}
\label{T2:GADesign}

\subsubsection{Solution Representation}
\label{T2:Representation}
Each candidate solution is encoded as a permutation of the city indices
$\{0, 1, \ldots, n{-}1\}$.  A chromosome such as $[3, 0, 7, 1, 5, 2, 6, 4]$
represents a tour that starts at city~3, visits city~0, then city~7, and so on,
returning to city~3 after visiting city~4.  The fitness of a chromosome is
computed as the total Euclidean tour length:
\[
f(\pi) = \sum_{i=0}^{n-1} d_{\pi_i,\,\pi_{(i+1) \bmod n}},
\]
where $\pi$ is the permutation and $d_{ij}$ is the distance between cities
$i$ and $j$.  A valid chromosome must contain each city exactly once;
violations of this property (duplicate or missing cities) render the
fitness value meaningless, as discussed in Section~\ref{T2:Infeasibility}.

\subsubsection{Population Initialisation}
\label{T2:Init}
The initial population is generated by creating $N$ random permutations of the
city indices, where $N$ is the population size.  Each permutation is produced
using a Fisher--Yates shuffle seeded by a NumPy random generator, ensuring
uniform sampling from the space of all $n!$ permutations.  No heuristic seeding
(e.g.\ nearest-neighbour tours) is used, so the initial population has maximum
diversity and the GA's convergence behaviour is attributable solely to the
evolutionary operators.

\subsubsection{Selection Mechanism}
\label{T2:Selection}
Two selection mechanisms are implemented in this study, namely tournament selection and roulette wheel selection. Both methods are designed to be swappable via a single configuration parameter, allowing direct comparison of their effect on the genetic algorithm's convergence behaviour.

\textbf{Tournament Selection}

Tournament selection operates by randomly sampling $k$ chromosomes from the current population and selecting the individual with the lowest tour length as the parent. In this study, $k = 3$ is used as the default tournament size. A higher value of $k$ increases selection pressure, causing the algorithm to favour fitter individuals more aggressively, while a lower value introduces more randomness and preserves population diversity.

Formally, given a population $P$ of size $N$ and a set of tournament candidates $C \subseteq P$ where $|C| = k$, the selected individual $s$ is defined as:

\begin{equation}
	s = \arg\min_{i \in C} f(i)
\end{equation}

where $f(i)$ denotes the tour length of individual $i$.

\textbf{Roulette Wheel Selection}

Roulette wheel selection assigns each chromosome a selection probability inversely proportional to its tour length. Since the TSP is a minimisation problem, shorter tours must receive higher selection probabilities. This is achieved by computing the inverse fitness for each individual and normalising the values to form a probability distribution.

Given a population of $N$ individuals with fitnesses $f_1, f_2, \ldots, f_N$, the selection probability of individual $i$ is:

\begin{equation}
	p_i = \frac{1/f_i}{\sum_{j=1}^{N} 1/f_j}
\end{equation}

Unlike tournament selection, roulette wheel selection allows weaker individuals to occasionally be selected, which can help maintain genetic diversity in the early stages of the search.

Both selection methods are wrapped in a unified \texttt{select} function that accepts a configuration parameter specifying the desired method. This strategy pattern allows the selection operator to be changed without modifying the GA loop, supporting the parameter sensitivity analysis conducted in Section~\ref{T3:Sensitivity}.

\subsubsection{Crossover Operators}
\label{T2:Crossover}
Three permutation-preserving crossover operators are implemented, each using
two randomly chosen cut points $p_1 < p_2$:

\textbf{Partially Mapped Crossover (PMX)}~\cite{goldberg1985alleles} copies
the segment $A[p_1 \ldots p_2]$ from Parent~A into the child, then resolves
conflicts from Parent~B by following a mapping chain until each displaced city
reaches an unoccupied position.  The remaining positions are filled directly
from Parent~B.

\textbf{Order Crossover (OX)}~\cite{davis1985applying} copies the same
segment from Parent~A, then scans Parent~B in wrap-around order starting
after $p_2$, inserting each city that does not already appear in the segment.
This preserves the relative ordering of Parent~B while inheriting a contiguous
block from Parent~A.

\textbf{Cycle Crossover (CX)}~\cite{oliver1987study} identifies cycles in the
position-wise mapping between the two parents.  Positions in each cycle are
inherited entirely from one parent, alternating parents across cycles.  Because
cycles partition the full set of positions, every city appears exactly once.

All three operators guarantee that the child is a valid permutation when both
parents are valid.  PMX is used as the default in the parameter sweep
(Section~\ref{T3:Sensitivity}).  The comparative experiment
(Section~\ref{T3:RepairCritique}) uses a naive single-point crossover that
deliberately breaks the permutation property, so that the effect of repair
and penalty strategies can be observed.

\subsubsection{Mutation Operators}
\label{T2:Mutation}
Swap mutation is implemented as the primary mutation operator.  Given a
chromosome and a mutation rate $r_m$, a random number is drawn uniformly
from $[0,1)$.  If the value is less than $r_m$, two positions $i$ and $j$
are selected uniformly at random and the cities at those positions are
exchanged.  Because swap mutation only rearranges existing cities, the
output is always a valid permutation.

Swap mutation makes a small, local perturbation (exactly two cities change
position), which helps fine-tune solutions in later generations without
disrupting large portions of the tour structure.

\subsection{Constraint Handling via Repair Mechanism (Task 2.2)}
\label{T2:Repair}

\subsubsection{Sources of Infeasibility}
\label{T2:Infeasibility}
A TSP chromosome is feasible only if it is a permutation of the city set. For an
$n$-city instance, every city in $\{0,1,\ldots,n-1\}$ must appear exactly
once; the return from the final city to the first is then added during fitness
evaluation. Infeasibility arises when crossover treats this chromosome as an
ordinary fixed-position string. A standard one-point, two-point, or uniform
crossover can copy values independently without checking whether a copied city
already appears elsewhere in the child. If one city is duplicated, another city
must be missing because the chromosome length is fixed.

Consider the two parents used in the crossover notebook:
\[
A=[0,1,2,3,4,5,6,7], \qquad
B=[2,6,4,0,5,7,1,3],
\]
with cut points at positions $2$ and $4$. A naive two-point crossover that starts
from $B$ and blindly overwrites the segment with $A_{2..4}=[2,3,4]$ produces
\[
[2,6,2,3,4,7,1,3].
\]
The violation can be seen position by position. At position 2, $B$'s value 4 is
replaced by 2, creating a second copy of city 2 because city 2 was already at
position 0. At position 3, $B$'s value 0 is replaced by 3, removing city 0 and
creating a second copy of city 3 because city 3 was already at position 7. At
position 4, $B$'s value 5 is replaced by 4, removing city 5. The resulting child
therefore duplicates cities 2 and 3 and misses cities 0 and 5. It has eight
entries, but it is not a Hamiltonian tour.

Permutation-aware crossover operators avoid this failure mode by adding logic
that preserves the city set. Partially Mapped Crossover (PMX) copies the selected
segment from one parent and uses the segment mapping to redirect displaced
values. In the example above, the copied segment creates mappings
$2\leftrightarrow4$, $3\leftrightarrow0$, and $4\leftrightarrow5$. After copying
$[2,3,4]$ into the child, value 0 from the second parent is routed through the
mapping until it reaches the open outside position 7, while value 5 is routed to
position 0. The remaining empty positions are filled from the second parent,
producing a valid child such as
\[
[5,6,2,3,4,7,1,0].
\]
The mapping chain is the key feasibility mechanism: every displaced city is sent
to a unique unoccupied position rather than being copied on top of an existing
city.

Order Crossover (OX) preserves feasibility in a different way. It first copies a
contiguous segment from the first parent, then scans the second parent in order
from just after the second cut point, skipping every value already present in
the copied segment. For the same parents, after copying $[2,3,4]$, the scan of
$B$ gives $7,1,3,2,6,4,0,5$. OX skips 3, 2, and 4 because they already appear in
the segment, leaving $[7,1,6,0,5]$. These are exactly the missing cities, so they
fill the remaining five positions starting at position 5 and wrapping around.
This produces
\[
[0,5,2,3,4,7,1,6].
\]
Thus OX combines the copied segment with its complement from the other parent;
no duplicate can be inserted because duplicates are explicitly skipped.

Cycle Crossover (CX) preserves feasibility by discovering cycles in the
position-wise mapping between the parents. Starting at position 0 in the example,
$B_0=2$, and city 2 is at position 2 in $A$; then $B_2=4$, city 4 is at position
4 in $A$; then $B_4=5$, city 5 is at position 5 in $A$; then $B_5=7$, city 7 is
at position 7 in $A$; then $B_7=3$, city 3 is at position 3 in $A$; finally
$B_3=0$, city 0 is at position 0 in $A$, returning to the start. This gives one
cycle of positions $(0,2,4,5,7,3)$; the remaining positions form the cycle
$(1,6)$. CX alternates which parent supplies whole cycles, for example producing
\[
[0,6,2,3,4,5,1,7].
\]
The cycles partition the chromosome positions, so every position is filled once
and the inherited values still form a complete permutation.

The downstream consequence of infeasible offspring is that their fitness values
are meaningless for the TSP. A chromosome with duplicate cities may skip an
expensive missing city and therefore appear to have an artificially short tour
length. Selection then treats this invalid distance as a legitimate objective
value, which corrupts selection pressure: tournament or roulette selection may
favour infeasible chromosomes, invalid genetic material can spread through the
population, and the search can converge toward low-cost strings that do not
represent tours. For this reason, the GA must either use feasibility-preserving
operators such as PMX, OX, and CX, or apply a repair mechanism before fitness
evaluation. The repair mechanism is especially important for naive crossover
baselines, implementation errors, and comparisons with alternative
constraint-handling strategies.

\subsubsection{Repair Mechanism Design}
\label{T2:RepairDesign}
The repair mechanism operates in two phases: \emph{diagnosis} and
\emph{correction}.

\textbf{Phase 1 --- Diagnosis.}  The chromosome is scanned left to right.
A set of seen cities is maintained; any city encountered a second time is
recorded as a duplicate (its position is flagged), and the set of all cities
not found in the chromosome is computed as the missing set.  For a chromosome
of length $n$, the number of duplicates always equals the number of missing
cities.

\textbf{Phase 2 --- Correction.}  Each duplicate position is replaced by a
city drawn from the missing set.  Three insertion strategies are implemented:

\begin{enumerate}[nosep]
    \item \textbf{Random insertion} --- missing cities are shuffled randomly
          and assigned to duplicate positions in order.  This is the fastest
          strategy and introduces no systematic bias.
    \item \textbf{Positional insertion} --- missing cities are sorted and
          paired with duplicate positions in index order.  Deterministic but
          does not consider distance.
    \item \textbf{Nearest-neighbour insertion} --- for each duplicate
          position, the missing city closest (by Euclidean distance) to the
          preceding city in the tour is selected.  This is the most
          expensive strategy but produces shorter post-repair tours.
\end{enumerate}

Random insertion is used as the default in all experiments because it avoids
introducing heuristic bias into the repair step, allowing the comparative
analysis to isolate the effect of repair itself rather than the quality of
the insertion heuristic.

The pseudocode below summarises the complete generational GA loop with
optional repair:

\begin{algorithm}[h]
\caption{Generational GA with Optional Repair}
\label{alg:ga}
\begin{algorithmic}[1]
\State Initialise population $P$ of $N$ random permutations
\State Evaluate fitness $f(p)$ for all $p \in P$
\For{$g = 1$ to $G_{\max}$}
    \State $P' \gets$ top-$e$ elites from $P$ \Comment{elitism}
    \While{$|P'| < N$}
        \State Select parents $p_1, p_2$ from $P$
        \If{$\text{rand}() < r_c$}
            \State $c \gets \text{Crossover}(p_1, p_2)$
        \Else
            \State $c \gets \text{copy}(p_1)$
        \EndIf
        \State $c \gets \text{Mutate}(c, r_m)$
        \If{repair enabled}
            \State $c \gets \text{Repair}(c)$
        \EndIf
        \State Add $c$ to $P'$
    \EndWhile
    \State $P \gets P'$; evaluate fitness for all $p \in P$
\EndFor
\State \Return best individual in $P$
\end{algorithmic}
\end{algorithm}

\subsubsection{Effect on Solution Quality, Diversity, and Convergence}
\label{T2:RepairEffect}
The repair mechanism is expected to influence the GA's behaviour in three ways.

\textbf{Solution quality.}  Without repair, infeasible chromosomes accumulate
in the population.  Their tour-length fitness is computed over an incomplete
set of cities, producing artificially low values that do not correspond to
valid tours.  Repair ensures that every chromosome evaluated by the GA is a
legitimate Hamiltonian tour, so fitness values are meaningful and comparable.

\textbf{Population diversity.}  Repair replaces duplicate cities with missing
ones, effectively injecting genetic material that would otherwise be lost.
Random insertion, in particular, introduces stochastic variation at the
repaired positions, which can help maintain diversity in the population.

\textbf{Convergence behaviour.}  By correcting infeasible offspring before
selection, repair prevents the population from converging toward low-fitness
but invalid solutions.  The search is kept within the feasible region of the
search space, so convergence reflects genuine improvement in tour quality
rather than accumulation of constraint violations.

These qualitative expectations are tested empirically in
Section~\ref{T3:RepairCritique}, where repair is compared against a penalty
function and an unrepaired baseline using 30 independent runs on kroA100.
